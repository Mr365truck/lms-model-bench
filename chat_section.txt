### Practical setup\
\
For each LM Studio config, record:\
\
| Metric | Why it matters |\
|---|---|\
| **TTFT** | How long before it starts answering |\
| **Output tok/s** | Raw generation speed |\
| **Total wall time** | What you actually feel |\
| **Pass rate** | Did the code work? |\
| **Edit quality** | Did it follow the requested change cleanly? |\
| **VRAM/RAM use** | Whether it is sustainable |\
\
For coding, I’d rank configs like this:\
\
**Pass rate \u003e correctness of edits \u003e total time \u003e tok/s**\
\
A faster config that subtly breaks code is worse.\
\
### Config matrix to test\
\
Start with this:\
\
| Test | Quant | Context | MTP |\
|---|---:|---:|---:|\
| A | Q4 / UD-Q4 | 16k | off |\
| B | Q4 / UD-Q4 | 16k | MTP 1 |\
| C | Q4 / UD-Q4 | 16k | MTP 2 |\
| D | Q4 / UD-Q4 | 16k | MTP 4 |\
| E | Q5 / Q6 if it fits | 16k | best MTP from above |\
| F | best quant | 32k | best MTP |\
\
Only change **one thing at a time**. For MTP, keep **parallel = 1** while testing.\

### Quick benchmark method\
\
In LM Studio:\
\
1. Load one config.\
2. Start the local server.\
3. Run the same 10–20 coding prompts.\
4. Save timing + whether the output passed.\
5. Change one setting and repeat.\
\
Use prompts like:\
\
```text\
Implement a Python LRUCache with get/put in O(1). Return only code.\
```\
\
```text\
Refactor this function to be clearer without changing behavior:\
\u003cpaste ugly function\u003e\
Return only the new function.\
```\
\
```text\
Find the bug in this code, explain it briefly, then provide the fixed code:\
\u003cpaste buggy code\u003e\
```\
\
```text\
Write pytest tests for this function:\
\u003cpaste function\u003e\
```\
\
### Better benchmark: use unit tests\
\
Make a folder of small tasks:\
\
```text\
bench/\
  task_01/\
    prompt.txt\
    tests.py\
  task_02/\
    prompt.txt\
    tests.py\
```\
\
For each config, ask the model to write `solution.py`, then run:\
\
```bash\
pytest tests.py\
```\
\
Score each task:\
\
```text\
1 = tests pass\
0 = tests fail\
```\
\
Then compare:\
\
```text\
Config C: 14/20 pass, 27 tok/s, 38 sec avg\
Config D: 13/20 pass, 35 tok/s, 31 sec avg\
```\
\
In that example, I’d choose **C** for coding unless D feels much better interactively.\
